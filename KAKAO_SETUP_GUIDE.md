# 카카오 로그인 설정 가이드

## 🚨 현재 해결된 문제
- `invalid_scope: account_email` 오류
- 카카오 로그인 scope 설정 문제

## 📋 카카오 개발자 콘솔 설정 단계

### 1. [카카오 개발자 콘솔](https://developers.kakao.com) 접속
- 내 애플리케이션 → 앱 선택

### 2. 플랫폼 설정
- **앱 설정** → **플랫폼** → **Web 플랫폼 추가**
- **도메인**: `http://localhost:3000`

### 3. 카카오 로그인 활성화
- **제품 설정** → **카카오 로그인** 
- **활성화 상태**: ON으로 설정

### 4. Redirect URI 설정 (현재 불필요)
- 현재 구현에서는 팝업 방식이므로 Redirect URI 설정 불필요
- 만약 redirect 방식으로 변경 시: `http://localhost:3000/auth/callback`

### 5. 동의항목 설정 (선택사항)
현재는 기본 권한만 사용하므로 추가 설정 불필요. 필요시:
- **제품 설정** → **카카오 로그인** → **동의항목**
- **닉네임**: 필수 동의
- **프로필 사진**: 선택 동의
- **카카오계정(이메일)**: 선택 동의 (필요시에만)

## 🔧 현재 설정 상태

### 환경 변수 (.env.local)
```env
# 카카오 로그인 설정
NEXT_PUBLIC_KAKAO_JS_KEY=3fc23201ea2d2318c1c8d6ecee1a2ef0
```

### 로그인 권한 요청
- ✅ **기본 권한**: 사용자 고유 ID
- ✅ **닉네임**: 자동으로 포함 (카카오 기본 정책)
- ✅ **프로필 사진**: 자동으로 포함 (카카오 기본 정책)
- ❌ **이메일**: 제거됨 (scope 오류 방지)

## 🎯 로그인 후 받을 수 있는 정보

```javascript
// 성공 시 받는 사용자 정보 예시
{
  id: 1234567890,
  properties: {
    nickname: "사용자닉네임",
    profile_image: "https://...",
    thumbnail_image: "https://..."
  },
  kakao_account: {
    // 이메일은 별도 권한 승인 필요
  }
}
```

## 🔄 추가 권한이 필요한 경우

이메일 등 추가 정보가 필요하다면:

1. **카카오 개발자 콘솔**에서 동의항목 설정
2. **앱 검수 신청** (선택 동의항목의 경우)
3. 코드에서 scope 추가:
```typescript
scope: 'profile_nickname,profile_image,account_email'
```

## ✅ 현재 상태
- 기본 카카오 로그인 정상 작동
- 사용자 ID, 닉네임, 프로필 사진 획득 가능
- scope 오류 해결완료
